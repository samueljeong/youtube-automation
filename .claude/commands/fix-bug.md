# 버그 수정 워크플로우

버그를 체계적으로 분석하고 수정합니다.

## 사용법

버그 설명을 인자로 전달하세요: `/fix-bug TTS 생성 시 타임아웃 발생`

## 워크플로우

### 1단계: 버그 분석
- 에러 로그 확인 (Render 대시보드 또는 로컬)
- 관련 코드 파일 식별
- 재현 조건 파악

### 2단계: 원인 추적
- 함수 호출 체인 추적
- 전역 변수 의존성 확인
- 에러 경로 분석

### 3단계: 수정 및 검증
- 최소한의 변경으로 수정
- 로컬 테스트 실행
- 코드 리뷰 에이전트 검토

### 4단계: 문서화
- docs/BUGFIX_CHANGELOG.md에 기록
- 커밋 메시지에 원인/해결 명시

## 체크리스트

- [ ] 에러 메시지 확인
- [ ] 관련 파일 읽기
- [ ] 수정 코드 작성
- [ ] 테스트 통과 확인
- [ ] BUGFIX_CHANGELOG.md 업데이트
- [ ] 커밋 및 푸시
